# So Rewrite Your System Spec Like This

## Universal AI Daily Publishing Architecture

**Content Model**
The platform is built around a “one item per day” publishing model. Each site defines what the daily item is (word, city, state, restaurant, etc.). The system generates one article per item per day using AI and live research, saves it as a draft, and requires admin approval before publishing.

**AI Pipeline**
When content is generated, the server performs live web research, builds a structured prompt, and sends it to OpenAI. The AI returns a structured JSON object containing title, summary, article body in markdown, highlights, and sources. This is saved as a draft in the database.

**Editorial Workflow**
All AI-generated content goes into a Draft queue. Nothing is published automatically. The admin reviews, edits, adds images, and approves content before it goes live.

**Public Site**
The public site shows the daily post, archive calendar, category filters, and individual article pages.

**Admin Panel**
The admin panel allows generating content, editing drafts, uploading images, publishing articles, and managing archives.

**Storage**
Images are stored directly in the database so they persist across deployments.
